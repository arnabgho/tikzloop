tikzloop: a verification sandbox & multi-turn environment for Text-to-TikZ

Implementation notes for the M3 Pro prototype, and a recipe for lifting the same loop into GRPO training on Google Colab.

base paper TikZilla (arXiv:2603.03072) repo ~/tinkering-projects/tikzloop status 11/11 tests passing date July 2026
Abstract TikZilla trains small LLMs to emit TikZ programs, rewarded by comparing the rendered figure against ground truth. We reproduce the executable core of that idea as a local prototype: a hardened compile-and-render sandbox (tectonic + poppler), a gym-style multi-turn environment whose observations are compiler errors and VLM critiques, and an inference loop (episode repair + best-of-N reranking) running entirely on Apple Silicon via MLX. Everything below the policy is pure CPU, so the identical sandbox becomes the reward function for GRPO-LoRA training on a free/cheap Colab GPU.

Architecture at a glance

The design principle: everything except the neural networks is deterministic, cheap, and portable. One episode of the environment looks like this —

Six modules, one job each:

  • sandbox.pyTikZ source → CompileResult{ok, log, image}. Tectonic in untrusted mode, command denylist, timeout, memory rlimit (Linux), capped 150 dpi render via pdftoppm. Auto-wraps bare tikzpictures, hoists preamble commands, injects packages/libraries it detects in the body.
  • env.pyTikZEnv.reset(description, gt) / .step(code) → StepResult. Termination on critic acceptance or turn budget. Pluggable Critic protocol and reward_fn.
  • policy.pyMLXPolicy wraps any mlx-lm chat model; build_messages threads (code, feedback) history into a repair conversation; extract_code parses fenced blocks. MockPolicy keeps tests model-free.
  • critic.pyVLMCritic (Qwen2.5-VL-3B-4bit) judges render vs. description, returns strict JSON {matches, issues[]} parsed into actionable feedback.
  • reward.pySigLIP patch embeddings + relaxed Earth Mover's similarity — a faithful stand-in for TikZilla's DeTikZify encoder EMD reward, swappable behind the same (img, gt) → [0,1] interface.
  • loop.pyrun_episode (multi-turn repair), best_of_n (sample-compile-rerank), JSONL episode logging — the raw material for hindsight-repair training data.

The verification sandbox

A TikZ sandbox is easier than a general code sandbox: the artifact is a picture, not a process. What we defend is the loop, against three failure modes of model-generated LaTeX:

Table 1. Threat model and mitigations, each covered by a test.
Failure modeExampleMitigationtest
Non-termination\loop\iftrue\repeatsubprocess timeout (25 s default)timeout ✓
Shell / filesystem access\write18{...}, \input--untrusted flag plus pre-compile regex denylistrejected ✓
Resource blowupgiant \foreach canvasRLIMIT_AS 2 GB (Linux), single page, 150 dpi cap, 2 kB log truncationcapped ✓
Design choice

Tectonic over TeX Live. A single ~50 MB binary versus a multi-GB apt install that eats 10+ minutes of every Colab session. Tectonic fetches LaTeX packages on demand into a cache directory — point XDG_CACHE_HOME at Google Drive and the download happens once, ever. Warm-cache compile time on the M3 Pro: 1.7–2.9 s per figure.

Making raw model output compilable

Models emit fragments: a bare tikzpicture, sometimes with \usetikzlibrary lines floating above it. ensure_standalone normalizes all of it — and this mirrors TikZilla's "dynamic package inclusion" data-cleaning step:

src/tikzloop/sandbox.py — normalizationpython
# 1. hoist preamble-only commands the model left in the body
hoisted = [m.group(0) for m in PREAMBLE_LINE.finditer(code)]   # \usetikzlibrary, \usepackage, ...

# 2. detect packages from environments        3. detect TikZ libraries from keys
r"\begin{circuitikz}" → \usepackage{circuitikz}     r"\b(diamond|star,)\b" → shapes.geometric
r"\begin{axis}"       → \usepackage{pgfplots}       r"\b(above|below)=.*\bof\b" → positioning
r"\begin{tikzcd}"     → \usepackage{tikz-cd}        r"-\{?(Latex|Stealth)"      → arrows.meta

# 4. wrap: \documentclass[tikz]{standalone} + preamble + body

This wasn't speculative hardening — the very first hand-written example (a flowchart with a diamond node) failed to compile until library detection existed. Compilation rate of the harness matters as much as compilation rate of the model.

Environment, policy, critic, reward

The environment contract

Deliberately gym-shaped, so it can back an inference demo today and a trainer tomorrow:

src/tikzloop/env.py — the whole APIpython
env = TikZEnv(critic=VLMCritic(), reward_fn=SigLIPReward(), max_turns=4)
env.reset(description, gt_image=gt)

step = env.step(code)
# StepResult(turn, code, compiled, feedback, done, image, reward, compile_seconds)
#   compiled=False → feedback is the truncated compiler log  ("fix the TikZ code")
#   compiled=True  → feedback is the critic's issue list; done when critic accepts

Two roles are kept strictly apart. The critic drives the loop — its feedback is the observation the policy conditions on next turn, and it needs no ground truth, so the loop runs on novel descriptions. The reward scores against ground truth — used for reranking, evaluation, and (on Colab) GRPO advantages, but never placed in the prompt. Blurring these is how reward hacking sneaks in.

Critic: a VLM as referee

Qwen2.5-VL-3B (4-bit, ~3.5 GB) sees the rendered PNG plus the target description and must answer in strict JSON: {"matches": bool, "issues": [...]}. Issues are phrased as actionable deltas ("node B should be below node A, it is to the right") because they get pasted verbatim into the repair prompt. A CompileOnlyCritic fallback accepts anything that compiles — that mode is the pure compiler-feedback repair loop, and it's what the tests exercise.

Reward: relaxed EMD over patch embeddings

TikZilla's reward is an Earth Mover's Distance between patch embeddings from a DeTikZify image encoder. Our prototype substitutes SigLIP (so400m-patch14-384 — DeTikZify's encoder is SigLIP-derived) and a relaxed EMD: match every patch to its best counterpart in the other image, average both directions. Same interface, one afternoon simpler; swap in the real DeTikZify-V2 encoder later without touching the env.

src/tikzloop/reward.py — the whole trickpython
def relaxed_emd_similarity(x, y):          # (n,d), (m,d) L2-normalized patches
    sim = x @ y.T
    return (sim.max(axis=1).mean() + sim.max(axis=0).mean()) / 2

Two inference harnesses

run_episode is Figure 1 as code — generate, step, append feedback, repeat until done. best_of_n is the test-time-scaling experiment: sample N independent programs at temperature 0.7, compile all, rerank by (compiled, reward). Both log to JSONL; every failed→fixed transition in those logs is a future training example (§5.4).

Cell 3 — the sandbox becomes the reward function

This is the payoff of the prototype: the reward function below is compile_tikz + relaxed_emd_similarity, verbatim. The paper's shaping is preserved — a small format floor for compiling at all, the semantic term on top, zero for anything that doesn't render.

In [3]:
GRPO reward: compile → render → embed → EMDpython
from concurrent.futures import ProcessPoolExecutor
from tikzloop.sandbox import compile_tikz
from tikzloop.policy  import extract_code
from tikzloop.reward  import SigLIPReward       # or the DeTikZify-V2 encoder, same interface

score = SigLIPReward(device="cuda")
pool  = ProcessPoolExecutor(max_workers=2)       # Colab gives ~2 cores; compilation is the CPU stage

def tikz_reward(prompts, completions, gt_image, **kw):
    codes   = [extract_code(c[0]["content"]) for c in completions]
    results = list(pool.map(compile_tikz, codes))             # parallel, sandboxed, timeout-safe
    return [0.0 if not r.ok
            else 0.1 + 0.9 * score(r.image, gt)               # format floor + semantic reward ∈ [0,1]
            for r, gt in zip(results, gt_image)]

Cell 4 — GRPO trainer, paper-faithful hyperparameters

In [4]:
Dr.GRPO variant, per TikZilla §5python
from trl import GRPOConfig, GRPOTrainer

config = GRPOConfig(
    num_generations      = 8,          # G rollouts per description (drop to 4 on T4)
    max_completion_length= 1024,
    temperature          = 1.0, top_p = 0.9,   # paper's sampling
    beta                 = 0.0,        # KL disabled, as in the paper

    loss_type            = "dr_grpo",  # token-level normalization, no length bias
    mask_truncated_completions = True, # paper: masking length-capped samples stabilizes training
    learning_rate        = 5e-6,
    per_device_train_batch_size = 2, gradient_accumulation_steps = 4,
    save_steps = 50, output_dir = "/content/drive/MyDrive/tikzloop/ckpt",
)
trainer = GRPOTrainer(model=model, args=config,
                      train_dataset=ds,               # description + gt_image columns (DaTikZ-V4 subset)

Where the multi-turn loop fits: hindsight repair

True trajectory-level multi-turn RL wants a vLLM server plus an env orchestrator — an A100-tier experiment. The budget path captures most of the value: run the inference loop (free, on the Mac or a T4), log every episode to JSONL, then explode each failed→fixed transition into a single-turn training example whose prompt contains the broken code and the verifier's feedback. Same VRAM as vanilla GRPO, and it directly optimizes the behavior the agentic loop needs — using feedback to repair.

Session survival rules

Assume every Colab session dies. (1) Checkpoint adapters to Drive/HF Hub every ~30 min (save_steps above — LoRA adapters are tens of MB). (2) Make cell 1 idempotent: mount, restore caches, resume_from_checkpoint=True. (3) Tectonic cache and HF cache live on Drive, not the VM disk. (4) Test the resume path before the long run — it's the difference between losing ten minutes and losing a day.

The experiment sequence this enables, cheapest first: (i) best-of-N reranking with the reward model (free, pure inference, already runnable), (ii) the agentic repair loop with a VLM critic (free), (iii) hindsight-repair GRPO on a T4 ($0), and (iv) the headline run — GRPO with a decomposed, verifiable reward on an A100 (≤ $50). Each stage produces the data and the baseline for the next.
